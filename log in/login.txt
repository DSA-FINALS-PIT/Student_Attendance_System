# VARIABLES USED IN LOGIN SYSTEM

1. DATABASE_FILE
   Purpose:
   Stores the filename of the account database.
   Value:
   "accounts.txt"

---

2. INFO_DATABASE_FILE
   Purpose:
   Stores the filename of the personal information database.
   Value:
   "personal_info.txt"

---

3. root
   Purpose:
   Main application window of the Tkinter program.

---

4. username_entry
   Purpose:
   Entry widget where the user types the username.

---

5. password_entry
   Purpose:
   Entry widget where the user types the password.

---

6. username
   Purpose:
   Temporarily stores the entered username during login or password recovery.

---

7. password
   Purpose:
   Temporarily stores the entered password during login.

---

8. found
   Purpose:
   Boolean variable used to check whether the username exists.

---

9. file
   Purpose:
   Used for opening, reading, and writing text files.

---

10. line
    Purpose:
    Stores one line from the text file while looping.

---

11. data
    Purpose:
    Stores separated values from a text file line using split(",").

---

12. stored_username
    Purpose:
    Stores the username retrieved from the database file.

---

13. stored_password
    Purpose:
    Stores the password retrieved from the database file.

---

14. role
    Purpose:
    Stores the selected or retrieved account role.
    Possible values:
    Student, Teacher, Admin

---

15. create_window
    Purpose:
    Secondary window used for creating accounts.

---

16. role_var
    Purpose:
    Tkinter StringVar used to track selected role.

---

17. form_frame
    Purpose:
    Frame widget that contains generated form fields.

---

18. entries
    Purpose:
    Dictionary that stores all Entry widgets.

---

19. widget
    Purpose:
    Temporary variable used when clearing old widgets.

---

20. fields
    Purpose:
    List containing field names for the selected role form.

---

21. field
    Purpose:
    Temporary variable representing one field name during looping.

---

22. entry
    Purpose:
    Temporary variable representing one Entry widget.

---

23. values
    Purpose:
    List containing all user input values before saving.

---

24. value
    Purpose:
    Stores the current text input from an Entry widget.

---

25. forgot_window
    Purpose:
    Window used for forgot password feature.

---

26. forgot_username
    Purpose:
    Entry widget where the user types username for password recovery.

========================================
VARIABLES INSIDE accounts.txt
=============================

Example Format:
StudentID,FullName,Course,YearSection,Gender,Birthdate,Contact,Email,Address,Username,Password,Role

---

values[0]
Purpose:
Stores ID Number

---

values[1]
Purpose:
Stores Full Name

---

values[2]
Purpose:
Stores Course/Program or Section/Position

---

values[3]
Purpose:
Stores Year & Section or Assigned Subjects

---

values[4]
Purpose:
Stores Gender or Contact Number

---

values[5]
Purpose:
Stores Birthdate or Email Address

---

values[6]
Purpose:
Stores Contact Number or Username

---

values[7]
Purpose:
Stores Email Address or Password

---

values[8]
Purpose:
Stores Address

---

values[9]
Purpose:
Stores Username

---

values[10]
Purpose:
Stores Password

---

data[-3]
Purpose:
Retrieves Username from database line.

---

data[-2]
Purpose:
Retrieves Password from database line.

---

data[-1]
Purpose:
